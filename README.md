# Web Player

Кастомный веб‑плеер на базе Video.js (mobile‑first), с поддержкой iOS/AirPlay,
выбором сериалов и эпизодов из локальных JSON, автопереходом серий, таймером сна,
сохранением сессии и удобными tap‑зонами

## Структура проекта

```
.
├── index.html
├── styles.css
├── app.js
└── data/
    ├── index.json           # манифест сериалов
    ├── office.json          # данные сериала
    └── allways_sunny.json   # данные сериала
```

## Локальный запуск

Плеер — статический сайт. Чтобы заработал `fetch()` для JSON, нужно поднять любой
локальный HTTP‑сервер:

```bash
# Python 3
python3 -m http.server 5500

# Node (npx)
npx serve .
```

Затем откройте [http://localhost:5500](http://localhost:5500).

> Открывать `index.html` напрямую через `file://` нельзя — браузер заблокирует
> загрузку JSON по CORS.

## Деплой на GitHub Pages

1. Создайте новый репозиторий и запушьте в него содержимое проекта.
2. В настройках репозитория `Settings → Pages` выберите ветку и папку `/` (root).
3. Через ~30 секунд получите ссылку вида `https://<user>.github.io/<repo>/`.

Все пути к JSON относительные (`data/...`), поэтому проект работает и в корне Pages,
и в подпапке.

## Добавление нового сериала

1. Положите JSON в `data/<slug>.json` в формате:

```json
[
  {
    "season": 1,
    "episodes": [
      { "id": "s01e01", "name": "Пилот", "url": "https://.../e01.mp4" }
    ]
  }
]
```

2. Добавьте сериал в `data/index.json`:

```json
{
  "defaultSlug": "office",
  "series": [
    { "slug": "my-show", "title": "My Show", "dataPath": "data/my-show.json" }
  ]
}
```

## Что реализовано

- **Video.js** с кастомизацией под бренд (тёмная тема, акцент, типографика Manrope).
- **Inline‑воспроизведение на iOS**: атрибуты `playsinline`, отключённые
  нативные tracks/controls.
- **Кастомные кнопки в control bar**:
  - Play/Pause (стандартная)
  - `« 10` — назад на 10 сек
  - `10 »` — вперёд на 10 сек
  - Прогресс
  - Текущее/общее время
  - Sleep countdown (отображается, когда таймер активен)
  - Sleep timer (10/20/30/60 / Off)
  - Autoplay next toggle
  - AirPlay (только iOS Safari, при доступности)
  - Fullscreen
- **Tap zones overlay**:
  - Левая 30% — `−10s`
  - Центр 40% — toggle play/pause
  - Правая 30% — `+10s`
  - Debounce 220 мс, визуальные индикаторы по бокам
- **Bottom sheet** выбора сериала на весь экран.
- **Аккордеон сезонов/эпизодов**, отметки «Сейчас» и «Просмотрено».
- **Sleep timer**: меню 10/20/30/60 + Off, обратный отсчёт в control bar,
  таймер не сбрасывается ручной паузой.
- **Autoplay next**: переход к следующей серии по `ended`. Если серий больше нет —
  оверлей «Сериал завершён».
- **SessionStore (localStorage)**: сериал, эпизод, позиция, autoplay‑настройка,
  громкость, мьют. После перезагрузки плеер открывает тот же эпизод и
  возвращается к позиции с погрешностью ~2 сек назад.
- **Error overlay + Retry**: первая попытка — перезагрузка `src`, при повторном
  фейле — полная переинициализация Video.js.
- **A11y**: `aria-label` для кастомных кнопок, фокус‑обводки, поддержка
  `prefers-reduced-motion`, минимальные touch‑targets ≥ 44×44.

## Известные ограничения iOS

- На iPhone в полноэкранном режиме Safari может показывать собственный системный
  плеер — это ограничение Webkit и обходится только частично. Inline‑режим
  работает корректно.
- AirPlay‑кнопка появляется только при доступности AirPlay
  (`webkitplaybacktargetavailabilitychanged`).
- Картинка в картинке (PiP) недоступна в PWA («На экран Домой») — ограничение
  WebKit; кнопка скрыта в standalone‑режиме. В Safari и на desktop PiP работает.

## Поддерживаемые браузеры

- iOS Safari 14+
- Android Chrome (последние версии)
- Desktop Chrome / Safari / Firefox / Edge (последние версии)
